# v-VSCS Privacy Policy

**Last Updated:** March 14, 2026

## 1. Introduction

This Privacy Policy describes how v-VSCS ("the Application") collects, uses, and handles your information. By installing and using v-VSCS, you agree to the terms described below.

## 2. Information We Collect

**Account Information:** If you use manage cloud profiles, we store a generic username and a hashed, randomly generated password.

**Configuration Profiles:** When you save profiles to the cloud, we store your facility ID, position configuration data, and the username associated with the profile.

**Connection Data:** When you connect to the v-VSCS server, we collect your socket connection ID, your IP address, and the facility/position you register with. Your IP address is accessible to the server as part of the underlying network connection but is not logged or stored. Session data is held in memory only for the duration of your connection and is not persisted.

**Audio Data:** Voice communication is transmitted peer-to-peer via WebRTC. Audio streams are not recorded, stored, or processed by our servers. Our server facilitates signaling (connection setup) only. Note that WebRTC peer-to-peer connections may expose your IP address to other connected clients as part of the standard WebRTC ICE negotiation process.

**Log Data:** The Application generates local log files on your device for troubleshooting purposes. These logs may contain connection events, errors, and diagnostic information. Log files remain on your device and are not transmitted to us.

## 3. How We Use Your Information

- To authenticate you for cloud profile management
- To facilitate voice communication between connected clients
- To manage and synchronize configuration profiles across sessions
- To deliver application updates

## 4. Data Storage and Security

Cloud profiles and account data are stored in a secured database. Passwords are hashed before storage. Session data (active connections, line states) is held in server memory only and is cleared when your session ends or the server restarts.

## 5. Data Sharing

We do not sell, trade, or share your personal information with third parties. Your voice communications are transmitted directly between peers and do not pass through or get stored on our servers.

## 6. Data Retention

Session data is not persisted beyond your active connection. Cloud profiles are retained until deleted.

## 7. Local Storage

The Application stores settings and local profiles on your device in the application data directory. You can access and delete these files at any time.

## 8. Updates

The Application checks for updates automatically. Update checks contact GitHub to determine if a new version is available. No personal data is transmitted during update checks.

## 9. Changes to This Policy

We may update this Privacy Policy from time to time. Continued use of the Application after changes constitutes acceptance of the revised policy.

## 10. Contact

If you have questions about this Privacy Policy, please open an issue on the project's GitHub repository.
